# KMeans Full Notes

## 核心公式

- 目标函数：$\min_{\{C_k\},\{\mu_k\}}\sum_k\sum_{x_i\in C_k}\lVert x_i-\mu_k\rVert^2$。
- 每轮 assignment/update 不增加该目标，但只能保证收敛到局部最优。

## 易错点

- 收敛判断放错位置，导致多跑一轮或提前停止。
- 空簇导致 `mean of empty slice`，必须定义处理策略。
- 初始化不同结果不同，工程中常跑多次取 inertia 最小的结果。
- 没有设置 `max_iter`，异常数据上可能运行过久。

## 面试追问

::: details 参考回答：KMeans 为什么只能保证局部最优？

KMeans 的目标函数对簇分配和中心同时优化时是非凸的，交替优化只能保证每一步不增大目标。它会收敛到某个稳定分配，但这个点不一定是全局最优。

:::

::: details 参考回答：KMeans++ 的初始化思想是什么？

KMeans++ 第一步随机选中心，之后按到已有中心的距离平方加权采样新中心。直觉是让初始中心分散覆盖数据空间，减少多个中心挤在同一簇附近的情况。

:::

::: details 参考回答：KMeans 适合非凸簇或不同密度簇吗？为什么？

KMeans 假设簇大致是凸的、球状的、方差接近的结构。非凸簇、不同密度簇或强噪声数据会让“离均值最近”这个准则失效，DBSCAN 或谱聚类可能更合适。

:::

::: details 参考回答：KMeans、GMM、DBSCAN 的核心假设有什么不同？

KMeans 用硬分配和均值中心，适合球状簇；GMM 用软分配和概率分布，可以表达椭圆形簇；DBSCAN 基于密度连接，可以发现任意形状簇并标记噪声。

:::

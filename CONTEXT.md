# ML Algorithm Practice Lab

本上下文定义这个个人学习项目中的核心术语。它只描述领域语言，不记录实现细节。

## Language

**手撕模块**:
一个围绕单个算法、训练机制或项目组件设计的可独立完成练习单元。
_Avoid_: 题目, 章节, notebook

**带提示版本**:
提供 TODO 分解、关键公式和实现路线的练习版本。
_Avoid_: 简单版, 教程版

**无提示版本**:
只给目标、输入输出约束和测试要求的练习版本。
_Avoid_: 困难版, 面试版

**项目拆解**:
把一个真实开源项目拆成可学习、可复现、可测试的多个手撕模块。
_Avoid_: 源码阅读, 项目笔记

**系统架构模块**:
解释项目如何从数据、训练、推理、服务和评估组成完整系统的学习模块。
_Avoid_: 架构图, 总结页

**题源线索**:
用于识别考点、题型和难度的外部公开信息，不等同于可直接收录的题目正文。
_Avoid_: 原题, 题库数据

**改写题面**:
基于题源线索重新组织的问题描述、输入输出和测试设定。
_Avoid_: 换数字, 搬运题

**手写实现约束**:
练习实现时只允许使用 Python 基础语法、list 和必要的 NumPy，不调用现成算法包。
_Avoid_: 调包, sklearn 实现, torch 内置层

## Relationships

- 一个 **手撕模块** 可以同时包含一个 **带提示版本** 和一个 **无提示版本**。
- 一个 **项目拆解** 由多个 **手撕模块** 和一个 **系统架构模块** 组成。
- 一个 **系统架构模块** 负责连接多个局部模块，而不是替代局部实现练习。
- 一个 **题源线索** 可以产生多个 **改写题面**。
- 一个 **手撕模块** 必须声明自己的 **手写实现约束**。

## Example dialogue

> **Dev:** "KMeans 要做成一个普通题目，还是一个手撕模块？"
> **Domain expert:** "做成手撕模块。第一次学看带提示版本，面试前做无提示版本，最后用 tests.py 验证。"
>
> **Dev:** "牛客题可以直接换数字收进来吗？"
> **Domain expert:** "不直接收原题。把它作为题源线索，重新写成自己的改写题面和测试。"

## Flagged ambiguities

- "新建一个文件" 在本次讨论中实际表示新建一个独立项目文件夹，并计划后续作为个人 GitHub 仓库维护。
- "换数字" 不作为独立合规标准；本项目采用 **改写题面**，要求题面叙述、数据设定和测试都重新组织。

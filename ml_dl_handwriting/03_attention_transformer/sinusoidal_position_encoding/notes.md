# Sinusoidal Position Encoding Notes

## 核心公式

- $PE_{pos,2i}=\sin(pos/10000^{2i/d_{model}})$。
- $PE_{pos,2i+1}=\cos(pos/10000^{2i/d_{model}})$。

## 易错点

- 指数写成 `i/d` 还是 `2i/d` 混乱，本实现按偶数维索引配对。
- 奇数 `d_model` 时 cos 维度切片要对齐。
- 位置编码要和 token embedding shape 对齐后相加。
- 绝对位置编码外推长度时不一定和训练分布一致。

## 面试追问

::: details 参考回答：Transformer 为什么需要位置编码？

attention 本身对 token 顺序近似置换不敏感，只知道内容相似度，不知道位置先后。位置编码给每个位置注入可区分信号，让模型能利用顺序和距离信息。

:::

::: details 参考回答：正弦位置编码为什么使用不同频率？

不同频率让位置在多个尺度上被编码，低频捕捉长距离变化，高频捕捉局部位置差异。类似用一组基函数表示位置，使模型更容易组合出相对距离模式。

:::

::: details 参考回答：绝对位置编码、可学习位置编码、RoPE 有什么区别？

绝对位置编码直接给每个位置一个向量，可学习版本灵活但外推弱。RoPE 把位置信息注入 Q/K 的旋转中，让 attention score 更自然地依赖相对位置；ALiBi 则直接给距离加线性偏置。

:::

::: details 参考回答：正弦位置编码为什么被认为具备一定长度外推能力？

正弦位置编码的公式可以为训练外的新位置继续计算，不依赖固定长度的参数表。它不保证外推一定好，但比固定可学习表更具备形式上的长度扩展能力。

:::
